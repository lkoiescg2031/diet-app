# Data Model

이 문서는 다이어트 앱의 초기 고수준 데이터 모델을 정의한다. 정확한 타입과 저장 API는 MVP 구현 spec과 plan에서 확정한다.

## 공통 원칙

- 모든 사용자 데이터는 브라우저 로컬에 저장한다.
- record는 고유 id를 가진다.
- 날짜는 일 단위 조회와 정렬이 쉬운 형식을 사용한다.
- 영속 데이터는 schema version을 가진다.
- 삭제보다 migration과 보존을 우선한다.

## 공통 필드 후보

| 필드 | 의미 |
| --- | --- |
| `id` | record 고유 식별자 |
| `date` | 사용자가 기록한 날짜 |
| `createdAt` | record 생성 시각 |
| `updatedAt` | 마지막 수정 시각 |
| `note` | 선택 메모 |

## ExerciseRecord

운동 기록 화면에서 사용한다.

필드 후보:

- 운동 이름
- 운동 종류
- 시간
- 강도
- 소모 칼로리 선택 입력
- 메모

## MealRecord

식단 기록 화면에서 사용한다.

필드 후보:

- 식사 구분
- 음식 이름
- 양 또는 serving
- 칼로리 선택 입력
- 단백질, 탄수화물, 지방 선택 입력
- 메모

## WeightRecord

체중 변화 기록 화면에서 사용한다.

필드 후보:

- 체중
- 단위
- 측정 날짜
- 메모

## OverviewData

전체 현황 화면은 저장 record를 직접 복제하지 않고 파생 계산으로 만든다.

계산 후보:

- 기간별 운동 횟수
- 기간별 총 운동 시간
- 기간별 식단 기록 수
- 기간별 칼로리 합계
- 최근 체중
- 체중 변화량

## 이후 결정 필요

- 정확한 날짜/시각 형식
- 칼로리와 영양소 입력 범위
- 체중 단위 기본값
- 통계 기간 기본값
- export/import schema
